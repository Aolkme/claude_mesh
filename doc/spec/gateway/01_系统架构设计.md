# 01_系统架构设计

> 本文档定义 Meshtastic 嵌入式网关管理系统的整体架构设计。

## 1. 系统概述

### 1.1 设计目标

| 目标 | 说明 |
|------|------|
| 服务常驻 | 后台守护进程持续运行，独立于客户端 |
| 多端控制 | CLI 和 Qt UI 都可控制同一服务 |
| 稳定可靠 | systemd 管理，异常自动重启 |
| 资源友好 | 适配嵌入式 ARM 平台，内存 < 10MB |

## 3. 核心模块设计

### 3.1 meshgateway 守护进程

```
┌─────────────────────────────────────────────────────────────────┐
│                      meshgateway daemon                         │
│                                                                 │
│  ┌──────────────┐  ┌──────────────┐  ┌──────────────────────┐  │
│  │ Main Thread  │  │Serial Thread │  │ Socket Server Thread │  │
│  │              │  │              │  │                      │  │
│  │ - Event Loop │  │ - Read/Write │  │ - Accept Connections │  │
│  │ - Config     │  │ - Frame Sync │  │ - Parse Commands     │  │
│  │ - Signal     │  │ - Callbacks  │  │ - Broadcast Events   │  │
│  └──────┬───────┘  └──────┬───────┘  └──────────┬───────────┘  │
│         │                 │                      │               │
│         └────────────────┬┴──────────────────────┘               │
│                          │                                       │
│                   ┌──────▼──────┐                                │
│                   │ Event Queue │                                │
│                   └──────┬──────┘                                │
│                          │                                       │
│  ┌───────────────────────▼───────────────────────────────────┐  │
│  │                    Core Engine                             │  │
│  │  ┌─────────────┐  ┌─────────────┐  ┌─────────────────┐    │  │
│  │  │Node Manager │  │Packet Parser│  │ Command Handler │    │  │
│  │  └─────────────┘  └─────────────┘  └─────────────────┘    │  │
│  └───────────────────────────────────────────────────────────┘  │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 3.2 线程模型

| 线程 | 数量 | 职责 |
|------|------|------|
| Main Thread | 1 | 事件循环、信号处理、配置管理 |
| Serial Thread | 1 | 串口读取、帧解析 |
| Socket Thread | 1 | 接受连接、处理请求 |
| Worker Pool | N | 异步任务处理 |

### 3.3 数据流

**接收数据流:**
```
Serial Device ──▶ Serial Thread ──▶ Frame Parser ──▶ Proto Parser
                                                              │
                                                              ▼
                    ┌─────────────────────────────────────────────┐
                    │              Event Queue                    │
                    └─────────────────────────────────────────────┘
                                      │
                    ┌─────────────────┼─────────────────┐
                    ▼                 ▼                 ▼
            Node Manager      Socket Clients       Log Writer
            (更新节点)         (广播事件)           (写日志)
```

**发送数据流:**
```
Socket Client ──▶ Command Parser ──▶ Command Handler
                                            │
                    ┌───────────────────────┼───────────────┐
                    ▼                       ▼               ▼
            Packet Builder          Node Manager    Config Manager
                    │
                    ▼
            Serial Thread ──▶ Serial Device
```

## 4. 进程通信设计

### 4.1 Unix Domain Socket

| 配置项 | 值 |
|--------|-----|
| 路径 | `/run/meshgateway.sock` |
| 权限 | 0666 (所有用户可连接) |
| 类型 | SOCK_STREAM (流式) |
| 最大连接 | 16 |
| 接收缓冲 | 64KB |
| 发送缓冲 | 64KB |

### 4.2 消息帧格式

```
┌────────────┬────────────┬──────────────────────┐
│ Length     │ Type       │ Payload              │
│ (4 bytes)  │ (1 byte)   │ (N bytes)            │
│ Big Endian │            │ JSON / Binary        │
└────────────┴────────────┴──────────────────────┘

Type:
  0x01 = JSON 请求/响应
  0x02 = JSON 事件推送
  0x03 = 二进制数据 (保留)
```

### 4.3 JSON 消息格式

**请求格式:**
```json
{
    "id": 12345,
    "cmd": "get_nodes",
    "params": {
        "node_id": "!aabbccdd"
    }
}
```

**响应格式:**
```json
{
    "id": 12345,
    "status": "ok",
    "data": {
        "nodes": [...]
    }
}
```

**事件推送格式:**
```json
{
    "event": "node_updated",
    "data": {
        "node_id": "!aabbccdd",
        "changes": ["position", "battery"]
    }
}
```

## 5. 配置管理

### 5.1 主配置文件

**路径:** `/opt/meshgateway/etc/meshgateway.conf`

```ini
[serial]
device = /dev/ttyUSB0
baudrate = 115200
reconnect_interval = 5

[network]
socket_path = /run/meshgateway.sock
max_clients = 16

[heartbeat]
interval = 600
timeout = 30

[log]
level = info
file = /opt/meshgateway/var/logs/meshgateway.log
max_size = 10
max_files = 5

[node]
expire_time = 7200
```

### 5.2 配置结构体

```c
typedef struct {
    /* Serial */
    char serial_device[256];
    uint32_t baudrate;
    uint32_t reconnect_interval;
    
    /* Network */
    char socket_path[256];
    uint32_t max_clients;
    
    /* Heartbeat */
    uint32_t heartbeat_interval;
    uint32_t heartbeat_timeout;
    
    /* Log */
    char log_level[16];
    char log_file[256];
    uint32_t log_max_size;
    uint32_t log_max_files;
    
    /* Node */
    uint32_t node_expire_time;
} config_t;

/* 加载配置 */
int config_load(config_t *cfg, const char *path);

/* 保存配置 */
int config_save(const config_t *cfg, const char *path);
```

### 6.1 错误码定义

```c
typedef enum {
    MG_OK = 0,
    
    /* 通用错误 (1-99) */
    MG_ERROR_UNKNOWN = 1,
    MG_ERROR_INVALID_PARAM = 2,
    MG_ERROR_NOT_CONNECTED = 3,
    MG_ERROR_TIMEOUT = 4,
    MG_ERROR_BUFFER_OVERFLOW = 5,
    
    /* 串口错误 (100-199) */
    MG_ERROR_SERIAL_OPEN = 100,
    MG_ERROR_SERIAL_READ = 101,
    MG_ERROR_SERIAL_WRITE = 102,
    MG_ERROR_SERIAL_FRAME = 103,
    MG_ERROR_SERIAL_TIMEOUT = 104,

    MG_ERROR_PROTO_DECODE = 200,
    MG_ERROR_PROTO_ENCODE = 201,
    MG_ERROR_UNKNOWN_CMD = 202,
    MG_ERROR_INVALID_JSON = 203,
    
    /* 节点错误 (300-399) */
    MG_ERROR_NODE_NOT_FOUND = 300,
    MG_ERROR_NODE_EXPIRED = 301,
    MG_ERROR_NODE_DB_ERROR = 302,
    
    /* Socket错误 (400-499) */
    MG_ERROR_SOCKET_CREATE = 400,
    MG_ERROR_SOCKET_BIND = 401,
    MG_ERROR_SOCKET_LISTEN = 402,
    MG_ERROR_SOCKET_ACCEPT = 403,
    MG_ERROR_SOCKET_CONNECT = 404,
    MG_ERROR_SOCKET_SEND = 405,
    MG_ERROR_SOCKET_RECV = 406,
    MG_ERROR_SOCKET_CLOSED = 407,
    
    /* 配置错误 (500-599) */
    MG_ERROR_CONFIG_LOAD = 500,
    MG_ERROR_CONFIG_SAVE = 501,
    MG_ERROR_CONFIG_PARSE = 502,
    
} mg_error_t;
```

### 6.2 错误响应格式

```json

    "id": 12345,
    "status": "error",
    "error": {
        "code": 300,
        "message": "Node not found",

}
```

## 7. 日志规范

### 7.1 日志级别

| 级别 | 宏 | 说明 | 使用场景 |
|------|-----|------|----------|
| DEBUG | LOG_DEBUG | 调试信息 | 帧解析细节、协议数据 |
| INFO | LOG_INFO | 常规信息 | 连接事件、节点更新 |
| WARN | LOG_WARN | 警告信息 | 重连尝试、配置缺失 |
| ERROR | LOG_ERROR | 错误信息 | 设备断开、解析失败 |

### 7.2 日志格式

```
[YYYY-MM-DD HH:MM:SS.mmm] [LEVEL] [MODULE] Message
```

**示例:**
```
[2024-01-15 10:30:45.123] [INFO] [serial] Connected to /dev/ttyUSB0
[2024-01-15 10:30:46.456] [DEBUG] [parser] Received frame: 94c3...
[2024-01-15 10:31:00.789] [WARN] [heartbeat] Heartbeat timeout, retrying...
[2024-01-15 10:31:05.012] [ERROR] [serial] Device disconnected
```

### 7.3 日志接口

```c
/* 日志初始化 */
int log_init(const char *file, const char *level, 
             uint32_t max_size, uint32_t max_files);

/* 日志输出 */
void log_debug(const char *module, const char *fmt, ...);
void log_info(const char *module, const char *fmt, ...);
void log_warn(const char *module, const char *fmt, ...);
void log_error(const char *module, const char *fmt, ...);

/* 日志关闭 */
void log_close(void);
```
