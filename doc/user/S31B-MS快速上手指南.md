## 一、产品概述

### 1.1 产品简介

S31B-MS 是 Thingseye 推出的 **Meshtastic Mesh 网络节点设备**，基于 STM32WLE5 芯片设计，专为物联网数据采集和无线组网通信而开发。

```
│  ┌─────────────┐    ┌─────────────┐    ┌─────────────┐            │
│  │  数据采集   │    │  Mesh 组网  │    │  远程管理   │            │
│  │             │    │             │    │             │            │
│  │ 传感器连接  │    │ 自动中继    │    │ 远程配置    │            │
│  │ 数据上报    │    │ 多跳通信    │    │ 固件升级    │            │
│  └─────────────┘    └─────────────┘    └─────────────┘            │
│                                                                     │
```

|------|------|

## 二、五分钟快速上手

> **目标**：从开箱到完成首次通信，仅需 5 分钟！

### 2.1 准备工作

```
快速上手检查清单：

□ S31B-MS 设备 × 1
□ LoRa 天线 × 1
□ 网关设备（推荐，首选方式）
□ 或手机蓝牙（通过 APP 连接，APP 开发中）
□ 或 USB-TTL 转换器（需拆开设备外壳，不推荐）
```

### 2.2 步骤一：安装天线

```
┌─────────────────────────────────────────────────────────────────┐
│                                                                 │
│   ⚠️  警告：请务必在通电前安装天线！                              │
│                                                                 │
│   无天线通电可能损坏射频模块！                                    │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

**操作步骤：**

1. 取出 LoRa 天线
2. 将天线拧紧到设备天线接口（SMA 接口）
3. 确保天线垂直向上
4. 天线周围无金属遮挡物

```
正确安装示意：

        ┌─────┐
        │  │  │  ← 天线垂直向上
        │  │  │
        │     │
     ┌──┴─────┴──┐
     │           │
     │  S31B-MS  │  ← 设备主体
     │           │
     └───────────┘
```

### 2.3 步骤二：设备上电

**供电方式：**

| 供电方式 | 说明 |
|----------|------|
| 太阳能供电 | 通过太阳能板供电，设备内置电池，支持长期户外部署 |

**上电操作：**

1. 确认天线已安装
2. 将设备放置在阳光充足的位置
3. 设备自动通过太阳能供电
4. 按下按键唤醒设备
5. 观察设备 LED 指示灯

### 2.4 步骤三：观察 LED 状态

按下设备按键，观察 LED 指示灯：

| LED 状态 | 含义 | 说明 |
|----------|------|------|
| 🔵 蓝色常亮 3 秒 | 设备启动中（未入网） | 新设备首次启动 |
| 🟢 绿色常亮 3 秒 | 正常工作中（已入网） | 设备已加入网络 |
| 🟢/🔵 闪烁 0.5 秒间隔 | 发送设备信息 | 长按按键 3 秒触发 |
| 🔴 红色闪烁 | 恢复出厂设置 | 长按按键 10 秒触发 |

```
LED 状态判断流程：

按下按键 ──→ 蓝色常亮 ──→ 设备未入网，需执行入网操作
    │
    └──→ 绿色常亮 ──→ 设备已入网，正常工作
```

### 2.5 步骤四：验证设备工作

**方式一：通过网关查看（推荐，首选）**

1. 启动 Thingseye 网关服务
2. 连接网关串口
3. 在 CLI 或 UI 中查看节点列表

```bash
# CLI 查看节点命令
./meshgate-cli nodes
```

**方式二：通过手机蓝牙（需配合 APP）**

1. 打开手机蓝牙
2. 启动 Thingseye APP（开发中）
3. 搜索并连接设备
4. 查看设备状态

**方式三：串口调试（需拆开设备，不推荐）**

1. 拆开设备外壳
2. 使用 USB-TTL 转换器连接设备串口（GND/TX/RX）
3. 打开串口终端软件（如 minicom、screen）
4. 设置波特率：**115200**
5. 观察设备输出的调试信息

## 四、网络部署

### 4.1 网络拓扑规划

```
Mesh 网络拓扑示意：

                        [ 网关节点 ]
                             │
              ┌──────────────┼──────────────┐
              │              │              │
         [ 节点A ] ◄────► [ 节点B ] ◄────► [ 节点C ]
              │              │              │
              │              │              │
         [ 节点D ] ◄────► [ 节点E ] ◄────► [ 节点F ]


Mesh 网络特点：
├── 节点上电自动发现邻居
├── 消息自动选择最优路径（多跳中继）
├── 单个节点故障不影响整体网络
├── 覆盖范围随节点数量增加而扩展
└── 支持数百个节点同时在线
```

### 4.2 网关部署

**网关位置选择：**

```
理想位置：
├── 网络覆盖区域的中心位置
├── 较高的安装位置（楼顶、塔架）
├── 电源供应稳定
├── 网络连接稳定（有线/WiFi/4G）
└── 天线周围无遮挡

避免位置：
├── 地下室、电梯井等封闭空间
├── 金属柜、金属房等屏蔽环境
├── 高压线、变电站等强电磁干扰区域
└── 大功率无线电设备附近
```

**网关安装步骤：**

1. 安装 LoRa 天线
2. 连接电源
3. 连接串口到主机（或使用内置串口）
4. 启动网关服务软件
5. 验证网关工作状态

### 4.3 节点部署

**节点位置选择：**

| 因素 | 建议 |
|------|------|
| 信号覆盖 | 确保与相邻节点有良好视距 |
| 安装高度 | 越高越好，建议 2 米以上 |
| 天线方向 | 垂直向上，避开遮挡 |
| 电源供应 | 根据功耗选择合适的供电方式 |

**节点安装步骤：**

1. 选择合适安装位置
2. 安装天线并调整方向
3. 确保太阳能板能接收到充足阳光
4. 按下按键唤醒设备
5. 观察 LED 状态确认启动

### 4.4 节点入网

> **入网**：将节点设备注册到 Mesh 网络，建立与管理员网关的信任关系。

#### 4.4.1 入网前提条件

```
入网检查清单：

□ 网关已部署并正常运行
□ 网关串口已连接
□ 节点设备已上电
□ 节点与网关在通信范围内
□ 节点处于未入网状态（蓝色LED）
```

#### 4.4.2 入网流程

**入网流程图：**

```
入网流程：

┌─────────────┐                    ┌─────────────┐
│   网关      │                    │   节点      │
│ (管理员)    │                    │ (未入网)    │
└──────┬──────┘                    └──────┬──────┘
       │                                  │
       │  1. 节点上电，广播 NodeInfo       │
       │ <─────────────────────────────────│
       │                                  │
       │  2. 网关收到节点信息              │
       │     节点出现在节点列表中          │
       │                                  │
       │  3. 发送入网邀请                  │
       │ ─────────────────────────────────>│
       │                                  │
       │                        ┌─────────┴─────────┐
       │                        │ 4. 节点存储       │
       │                        │    公司公钥       │
       │                        │    网关公钥       │
       │                        │    标记已入网     │
       │                        └─────────┬─────────┘
       │                                  │
       │  5. 入网成功响应                  │
       │ <─────────────────────────────────│
       │                                  │
       │  ✓ 入网完成                       │
```

**入网流程说明：**

**步骤 1：节点上电启动**

节点设备安装天线后，放置在阳光充足的位置，太阳能板为设备供电。按下按键唤醒设备，LED 显示蓝色表示设备处于未入网状态。

**步骤 2：节点广播信息**

节点启动后，会自动向 Mesh 网络广播 NodeInfo 消息，包含节点的 ID、名称、公钥等信息。此过程为自动执行，无需用户干预。

**步骤 3：网关发现节点**

网关收到节点的 NodeInfo 消息后，将节点信息添加到节点列表中。用户可通过 CLI 命令或 Web UI 查看到新发现的节点，此时节点状态显示为「未入网」。

**步骤 4：发送入网邀请**

用户在网关的 CLI 或 UI 界面中，选择未入网的节点，点击「入网」按钮或执行入网命令。网关将 SetCompanyKeyRequest 消息发送给节点，消息中包含：公司公钥、网关公钥、网关节点 ID、时间戳等信息。

**步骤 5：节点处理入网请求**

节点收到入网邀请后，执行以下操作：
1. 验证消息有效性
2. 存储公司公钥到本地
3. 存储网关公钥，将网关设置为管理员
4. 将 is_enrolled 标记为 true
5. 保存配置到非易失性存储器

**步骤 6：入网成功**

节点向网关返回入网成功响应消息，网关更新节点状态为「已入网」。此时按下节点按键，LED 显示绿色，表示设备已成功入网，可正常使用。

#### 4.4.3 入网操作（CLI 方式）

**步骤一：启动网关服务**

```bash
cd /path/to/meshgateway
./meshgateway -c meshgateway.conf
```

**步骤二：连接串口**

```bash
./meshgate-cli connect --device /dev/ttyUSB0
```

**步骤三：查看节点列表**

```bash
./meshgate-cli nodes
```

输出示例：
```
节点 ID        长名称      短名称   电量   已入网
!aabbccdd     设备001     D01      85%    否
!11223344     设备002     D02      92%    是
```

**步骤四：发送入网邀请**

```bash
# 入网单个节点
./meshgate-cli enroll-device --node !aabbccdd

# 或批量入网所有未入网节点
./meshgate-cli enroll-all
```

**步骤五：验证入网成功**

```bash
./meshgate-cli node --id !aabbccdd
```

查看输出中 `已入网` 字段是否为 `是`。

#### 4.4.4 入网操作（UI 方式）

1. 打开浏览器访问网关地址（默认 `http://localhost:8080`）
2. 在 Dashboard 页面连接串口
3. 切换到 Nodes 页面查看节点列表
4. 点击未入网节点的「入网」按钮
5. 等待入网完成，节点状态变为「已入网」

### 4.5 节点角色选择

| 角色 | 适用位置 | 说明 |
|------|----------|------|
| **CLIENT** | 网络中心/中间位置 | 参与消息转发，扩展网络覆盖 |
| **CLIENT_MUTE** | 网络边缘 | 不转发其他设备消息，减少网络负载 |

```
节点角色配置建议：

小型网络（<20节点）：
├── CLIENT: 90%
└── CLIENT_MUTE: 10%

中型网络（20-50节点）：
├── CLIENT: 80-85%
└── CLIENT_MUTE: 15-20%

大型网络（>50节点）：
├── CLIENT: 70-80%
└── CLIENT_MUTE: 20-30%
```

### 4.6 网络验证

**验证网络连通性：**

```bash
# 查看所有节点状态
./meshgate-cli nodes

# 向指定节点发送测试消息
./meshgate-cli send --to !aabbccdd "测试消息"

# 查看实时事件
```

## 五、设备配置

### 5.1 配置方式

| 方式 | 适用场景 | 说明 |
|------|----------|------|
| **CLI 命令行** | 批量配置、自动化脚本 | 功能完整，适合专业人员 |
| **Web UI 界面** | 日常操作、可视化管理 | 图形界面，操作直观 |

### 5.2 基础配置项

| 配置项 | 说明 | 默认值 |
|------|------|--------|
| **设备名称** | 节点在网络中的名称 | Meshtastic_xxxx |
| **短名称** | 4 字符缩写，用于显示 | xxxx |
| **区域** | LoRa 频段，需符合当地法规 | CN（470-510MHz） |
| **角色** | 节点在网络中的角色 | CLIENT |

### 5.3 私有信道配置

私有信道用于隔离不同 Mesh 网络，确保通信安全。

| 参数 | 说明 | 要求 |
|------|------|------|
| **信道名称** | 网络标识符 | 同一网络内所有节点必须一致 |
| **信道密钥** | 加密密钥（32字节） | 同一网络内必须一致，建议系统生成 |
| **区域频段** | LoRa 工作频段 | 需符合当地无线电法规 |

```
私有信道配置步骤：

1. 在网关配置好私有信道
2. 节点入网后，网关自动下发信道配置
3. 节点接收配置并重启
4. 节点加入新的私有网络
```

> **注意**：修改信道配置后，节点将重启并加入新网络，原网络连接将断开。

### 5.4 低功耗配置

S31B-MS 支持智能睡眠策略，实现长时间续航。

#### 5.4.1 工作模式

```
节点工作周期：

   睡眠              唤醒工作              睡眠
   ├────────────────┤├──────────────────┤├────────────────┤
   
   ←── 唤醒间隔 ────→←── 唤醒时长 ──────→
```

#### 5.4.2 配置参数

| 参数 | 说明 | 建议范围 |
|------|------|----------|
| **唤醒间隔** | 两次唤醒之间的时间间隔 | 1分钟 ~ 24小时 |
| **唤醒时长** | 每次唤醒后持续工作时间 | 1分钟 ~ 30分钟 |
| **分时策略** | 按时段设置不同唤醒频率 | 可选 |

#### 5.4.3 功耗估算

## 七、故障排查

### 7.1 常见问题

#### 问题1：设备无法上电

**现象**：设备 LED 不亮

**排查步骤**：

1. 检查太阳能板是否连接正常
2. 检查太阳能板是否被遮挡，确保阳光充足
3. 检查设备内置电池是否有电
4. 将设备放在阳光下充电一段时间后重试
5. 联系技术支持

#### 问题2：串口连接失败

**现象**：CLI 连接串口时报错

**排查步骤**：

```bash
# 检查设备是否存在
ls /dev/ttyUSB*

# 检查用户权限（需要 dialout 组）
groups

# 添加用户到 dialout 组
sudo usermod -aG dialout $USER
# 添加后需要重新登录
```

#### 问题3：节点无法发现

**现象**：网关节点列表为空

**排查步骤**：

1. 确认节点已上电且 LED 正常
2. 确认节点与网关距离在通信范围内
3. 确认节点和网关使用相同频段
4. 尝试按下节点按键触发广播

#### 问题4：入网失败

**现象**：发送入网邀请后节点状态未变化

**排查步骤**：

1. 确认节点处于未入网状态（蓝色 LED）
2. 确认节点与网关通信正常
3. 查看网关日志获取错误信息
4. 尝试重置节点后重新入网

### 7.2 LED 状态速查表

| LED 状态 | 含义 | 处理建议 |
|----------|------|----------|
| 🔵 蓝色常亮 3 秒 | 未入网状态 | 执行入网操作 |
| 🟢 绿色常亮 3 秒 | 已入网，正常工作 | 无需处理 |
| 🟢/🔵 闪烁 | 正在发送信息 | 等待完成 |
| 🔴 红色闪烁 | 恢复出厂设置 | 重新配置 |
| 无 LED | 未上电或故障 | 检查电源 |
