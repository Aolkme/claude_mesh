## 一、部署规划

### 1.1 部署前准备

```
部署检查清单：

□ 明确网络覆盖范围
□ 确定节点数量和位置
□ 规划网关位置
□ 准备供电方案
□ 准备安装工具和材料
□ 了解当地无线电管理规定
```

### 1.2 需求分析

| 需求项 | 分析要点 |
|--------|----------|
| **覆盖范围** | 需要覆盖的地理区域面积 |
| **节点密度** | 单位面积内的节点数量 |
| **数据频率** | 数据采集和上报频率 |
| **供电条件** | 是否有稳定电源或需要电池/太阳能 |
| **网络接入** | 网关如何连接互联网（有线/WiFi/4G） |

### 1.3 频段选择

| 区域 | 频段 | 说明 |
|------|------|------|
| 中国 | 470-510 MHz | 需要型号核准 |
| 欧洲 | 868 MHz | CE 认证 |
| 美国 | 915 MHz | FCC 认证 |
| 其他 | 请咨询当地法规 | - |

## 二、网络拓扑设计

### 2.1 拓扑类型

```
类型一：星型拓扑

                    [网关]
                   /  |  \
                  /   |   \
             [节点] [节点] [节点]

适用场景：小范围、节点少


类型二：链型拓扑

[网关] ─── [节点] ─── [节点] ─── [节点]

适用场景：线性区域（道路、河流沿线）


类型三：网状拓扑（推荐）

                    [网关]
                   /  |  \
                  /   |   \
             [节点]─[节点]─[节点]
                |  \  |  /  |
             [节点]─[节点]─[节点]

适用场景：大面积、多节点
```

### 2.2 节点角色规划

| 角色 | 数量建议 | 适用位置 |
|------|----------|----------|
| **CLIENT** | 70-90% | 网络中心、中继路径上 |
| **CLIENT_MUTE** | 10-30% | 网络边缘、叶子节点 |

### 2.3 容量规划

| 网络规模 | 节点数量 | 网关数量 | 建议 |
|----------|----------|----------|------|
| 小型 | < 20 | 1 | 单网关即可 |
| 中型 | 20-50 | 1-2 | 可增加备用网关 |
| 大型 | 50-200 | 2-5 | 多网关负载均衡 |
| 超大型 | > 200 | 5+ | 分区部署 |

## 三、网关部署

### 3.1 位置选择

```
理想位置：
├── 网络覆盖区域中心
├── 较高位置（楼顶、塔架）
├── 电源稳定
├── 网络连接稳定
└── 天线无遮挡

避免位置：
├── 地下室、电梯井
├── 金属封闭空间
├── 高压线附近
└── 强电磁干扰区域
```

### 3.2 安装步骤

1. **天线安装**
   - 安装外置 LoRa 天线
   - 确保天线垂直
   - 避免金属遮挡

2. **电源连接**
   - 连接稳定电源
   - 建议配置 UPS

3. **网络连接**
   - 连接有线网络（推荐）
   - 或配置 WiFi/4G

4. **服务启动**
   ```bash
   ./meshgateway -c meshgateway.conf
   ```

### 3.3 配置示例

```ini
# meshgateway.conf

[serial]
device = /dev/ttyUSB0
baudrate = 115200

[network]
host = 127.0.0.1
port = 9999

[web]
bind = 0.0.0.0:8080
static_dir = static

[log]
level = info
file = meshgateway.log
```

## 四、节点部署

### 4.1 位置选择原则

| 原则 | 说明 |
|------|------|
| **视距通信** | 节点之间尽量保持视距 |
| **高度优先** | 安装位置越高越好 |
| **避免遮挡** | 天线周围无金属遮挡 |
| **供电方便** | 靠近电源或适合电池安装 |

### 4.2 安装步骤

1. **检查设备**
   - 确认设备完好
   - 确认固件版本

2. **安装天线**
   - ⚠️ 必须在通电前安装
   - 拧紧天线，确保垂直

3. **连接电源**
   - 连接 USB/电池/太阳能
   - 观察 LED 状态

4. **唤醒设备**
   - 按下按键唤醒
   - 确认 LED 状态正常

### 4.3 供电方案

| 方案 | 适用场景 | 说明 |
|------|----------|------|
| **USB 供电** | 测试、固定位置 | 简单方便 |
| **电池供电** | 无电源区域 | 需定期更换 |
| **太阳能+电池** | 户外长期部署 | 建议配置 |

## 五、网络验证

### 5.1 连通性测试

```bash
# 查看节点列表
./meshgate-cli nodes

# 发送测试消息
./meshgate-cli send --to !aabbccdd "测试消息"

# 实时监控
./meshgate-cli monitor
```

### 5.2 信号质量检测

| 指标 | 说明 | 良好值 |
|------|------|--------|
| **SNR** | 信噪比 | > -5 dB |
| **RSSI** | 接收信号强度 | > -100 dBm |
| **跳数** | 消息经过的节点数 | < 3 跳 |

### 5.3 网络覆盖率验证

```
验证方法：

1. 在覆盖区域边缘测试通信
2. 检查所有节点是否都能与网关通信
3. 验证消息延迟是否满足要求
4. 记录信号质量数据
```

## 六、运维管理

### 6.1 日常巡检

| 检查项 | 频率 | 内容 |
|--------|------|------|
| 节点在线率 | 每日 | 检查节点是否在线 |
| 电池电量 | 每周 | 检查电池供电节点电量 |
| 网关状态 | 每日 | 检查网关运行状态 |
| 日志检查 | 每周 | 检查错误日志 |

### 6.2 故障处理流程

```
故障发现 ─→ 问题定位 ─→ 故障分类 ─→ 处理措施 ─→ 验证恢复

常见故障：
├── 节点离线：检查电源、信号
├── 通信异常：检查信道配置
├── 网关故障：检查日志、重启服务
└── 数据丢失：检查网络稳定性
```

### 6.3 固件升级

```bash
# 远程升级节点固件
./meshgate-cli admin upgrade --node !aabbccdd --fw firmware.bin
```
