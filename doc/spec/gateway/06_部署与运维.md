# 06_部署与运维

> 本文档定义 meshgateway 系统的部署、编译和运维指南。

## 1. 系统要求

### 1.1 硬件要求

| 平台 | CPU | 内存 | 存储 |
|------|-----|------|------|
| ARM 嵌入式 | ARMv7+ | 128MB+ | 64MB+ |
| x86 服务器 | x86_64 | 256MB+ | 128MB+ |

### 1.2 软件依赖

**编译依赖:**
```
- GCC 7+ / Clang 6+
- CMake 3.10+
- Make
- protobuf-c / nanopb
- systemd开发库
```

**运行依赖:**
```
- systemd
- glibc 2.17+
- protobuf运行库
```

**UI 依赖 (可选):**
```
- Qt 5.15+ 或 Qt 6.x
```

## 2. 编译指南

### 2.1 获取源码

```bash
git clone https://github.com/your-org/meshgateway.git
cd meshgateway
git submodule update --init --recursive
```

### 2.2 编译核心服务

```bash
mkdir build && cd build
cmake -DCMAKE_BUILD_TYPE=Release ..
make -j$(nproc)
```

### 2.3 编译 CLI 工具

```bash
cd cli
mkdir build && cd build
cmake -DCMAKE_BUILD_TYPE=Release ..
make -j$(nproc)
```

### 2.4 编译 Qt UI

```bash
cd ui
mkdir build && cd build
cmake -DCMAKE_BUILD_TYPE=Release ..
make -j$(nproc)
```

### 2.5 交叉编译 (ARM)

```bash
# 设置交叉编译工具链
export CC=arm-linux-gnueabihf-gcc
export CXX=arm-linux-gnueabihf-g++

cmake -DCMAKE_BUILD_TYPE=Release \
      -DCMAKE_TOOLCHAIN_FILE=../cmake/arm-linux-gnueabihf.cmake \
      ..
make -j$(nproc)
```

## 3. 安装部署

### 3.1 手动安装

```bash
# 创建用户
sudo useradd -r -s /bin/false meshgate

# 创建目录
sudo mkdir -p /opt/meshgateway/{bin,lib,etc,var/logs,run}

# 安装二进制
sudo install -m 755 build/meshgateway /opt/meshgateway/bin/
sudo install -m 755 cli/build/meshgate-cli /opt/meshgateway/bin/
sudo install -m 755 ui/build/meshgate-ui /opt/meshgateway/bin/

# 安装配置
sudo install -m 644 config/meshgateway.conf /opt/meshgateway/etc/
sudo install -m 644 config/nodes.conf /opt/meshgateway/etc/

# 安装 systemd 服务
sudo install -m 644 systemd/meshgateway.service /etc/systemd/system/

# 设置权限
sudo chown -R meshgate:meshgate /opt/meshgateway/var
sudo chmod 750 /opt/meshgateway/var
```

### 3.2 Debian/Ubuntu 包

```bash
# 构建 deb 包
dpkg-buildpackage -us -uc

# 安装
sudo dpkg -i meshgateway_1.0.0_arm64.deb
```

### 3.3 RPM 包

```bash
# 构建 rpm 包
rpmbuild -ba packaging/meshgateway.spec

# 安装
sudo rpm -i meshgateway-1.0.0-1.el7.x86_64.rpm
```

## 4. systemd 服务管理

### 4.1 服务文件

**路径:** `/etc/systemd/system/meshgateway.service`

```ini
[Unit]
Description=Meshtastic Gateway Service
Documentation=https://github.com/your-org/meshgateway
After=network.target network-online.target
Wants=network-online.target

[Service]
Type=notify
NotifyAccess=all
ExecStart=/opt/meshgateway/bin/meshgateway
ExecReload=/bin/kill -HUP $MAINPID
Restart=on-failure
RestartSec=5
TimeoutStartSec=30
TimeoutStopSec=10

User=meshgate
Group=meshgate
WorkingDirectory=/opt/meshgateway

# 环境变量
Environment=MG_CONFIG_PATH=/opt/meshgateway/etc/meshgateway.conf

# 资源限制
LimitNOFILE=1024
MemoryMax=50M
CPUQuota=25%

# 安全加固
NoNewPrivileges=yes
ProtectSystem=strict
ProtectHome=yes
ReadWritePaths=/opt/meshgateway/var /run
PrivateTmp=yes
ProtectKernelTunables=yes
ProtectControlGroups=yes

[Install]
WantedBy=multi-user.target
```

### 4.2 服务操作

```bash
# 重载服务文件
sudo systemctl daemon-reload

# 启动服务
sudo systemctl start meshgateway

# 停止服务
sudo systemctl stop meshgateway

# 重启服务
sudo systemctl restart meshgateway

# 查看状态
sudo systemctl status meshgateway

# 开机自启
sudo systemctl enable meshgateway

# 禁用自启
sudo systemctl disable meshgateway

# 查看日志
sudo journalctl -u meshgateway -f
```

### 4.3 服务模板 (多实例)

```ini
# /etc/systemd/system/meshgateway@.service
[Unit]
Description=Meshtastic Gateway Instance %i
After=network.target

[Service]
Type=notify
ExecStart=/opt/meshgateway/bin/meshgateway -c /opt/meshgateway/etc/meshgateway-%i.conf
User=meshgate
Group=meshgate

[Install]
WantedBy=multi-user.target
```

```bash
# 启动实例
sudo systemctl start meshgateway@ttyUSB0
sudo systemctl start meshgateway@ttyUSB1
```

## 5. 配置管理

### 5.1 主配置文件

**路径:** `/opt/meshgateway/etc/meshgateway.conf`

```ini
[serial]
# 串口设备路径
device = /dev/ttyUSB0
# 波特率
baudrate = 115200
# 重连间隔 (秒)
reconnect_interval = 5

[network]
# Unix Socket 路径
socket_path = /run/meshgateway.sock
# 最大客户端连接数
max_clients = 16

[heartbeat]
# 心跳间隔 (秒)
interval = 600
# 心跳超时 (秒)
timeout = 30

[log]
# 日志级别: debug, info, warn, error
level = info
# 日志文件路径
file = /opt/meshgateway/var/logs/meshgateway.log
# 单个日志文件最大大小 (MB)
max_size = 10
# 保留日志文件数量
max_files = 5

[node]
# 节点过期时间 (秒)
expire_time = 7200
```

### 5.2 配置验证

```bash
# 检查配置语法
meshgate-cli config --check

# 查看当前配置
meshgate-cli config
```

### 5.3 运行时重载

```bash
# 发送 SIGHUP 信号重载配置
sudo systemctl reload meshgateway
# 或
sudo kill -HUP $(cat /run/meshgateway.pid)
```

## 6. 日志管理

### 6.1 日志位置

| 日志类型 | 路径 |
|----------|------|
| 服务日志 | `/opt/meshgateway/var/logs/meshgateway.log` |
| systemd日志 | `journalctl -u meshgateway` |
| 内核日志 | `dmesg | grep -i usb` |

### 6.2 日志轮转

**路径:** `/etc/logrotate.d/meshgateway`

```
/opt/meshgateway/var/logs/*.log {
    daily
    rotate 7
    compress
    delaycompress
    missingok
    notifempty
    create 0640 meshgate meshgate
    postrotate
        systemctl reload meshgateway > /dev/null 2>&1 || true
    endscript
}
```

### 6.3 日志查看

```bash
# 实时查看
tail -f /opt/meshgateway/var/logs/meshgateway.log

# 查看 systemd 日志
journalctl -u meshgateway -f

# 按时间过滤
journalctl -u meshgateway --since "2024-01-15 10:00:00" --until "2024-01-15 11:00:00"

# 按级别过滤
journalctl -u meshgateway -p err
```

## 7. 监控与诊断

### 7.1 健康检查脚本

```bash
#!/bin/bash
# /opt/meshgateway/bin/health_check.sh

SOCKET="/run/meshgateway.sock"
TIMEOUT=5

# 检查服务是否运行
if ! systemctl is-active --quiet meshgateway; then
    echo "CRITICAL: Service not running"
    exit 2
fi

# 检查 Socket 是否存在
if [ ! -S "$SOCKET" ]; then
    echo "CRITICAL: Socket not found"
    exit 2
fi

# 检查服务响应
RESPONSE=$(timeout $TIMEOUT meshgate-cli status --json 2>/dev/null)
if [ $? -ne 0 ]; then
    echo "CRITICAL: Service not responding"
    exit 2
fi

# 检查串口状态
SERIAL_STATE=$(echo "$RESPONSE" | jq -r '.data.serial_state')
if [ "$SERIAL_STATE" != "connected" ]; then
    echo "WARNING: Serial not connected"
    exit 1
fi

echo "OK: Service healthy"
exit 0
```

## 8. 故障排查

### 8.1 常见问题

#### 服务无法启动

```bash
# 检查日志
journalctl -xeu meshgateway

# 检查配置
meshgate-cli config --check

# 检查权限
ls -la /opt/meshgateway/
ls -la /run/meshgateway.sock
```

#### 串口连接失败

```bash
# 检查设备是否存在
ls -la /dev/ttyUSB*

# 检查设备权限
groups meshgate
# 确保用户在 dialout 组
sudo usermod -a -G dialout meshgate

# 检查设备占用
lsof /dev/ttyUSB0
```

#### Socket 连接失败

```bash
# 检查 Socket 文件
ls -la /run/meshgateway.sock

# 检查服务状态
systemctl status meshgateway

# 测试连接
meshgate-cli status
```

### 8.2 调试模式

```bash
# 启用调试日志
meshgate-cli config --set log.level=debug
sudo systemctl reload meshgateway

# 前台运行 (调试)
sudo /opt/meshgateway/bin/meshgateway -D
```

### 8.3 性能分析

```bash
# 检查内存使用
ps aux | grep meshgateway

# 检查 CPU 使用
top -p $(pgrep meshgateway)

# 检查文件描述符
lsof -p $(cat /run/meshgateway.pid) | wc -l
```

## 9. 备份与恢复

### 9.1 备份

```bash
#!/bin/bash
# /opt/meshgateway/bin/backup.sh

BACKUP_DIR="/opt/meshgateway/var/backups"
DATE=$(date +%Y%m%d_%H%M%S)

mkdir -p "$BACKUP_DIR"

# 备份配置
tar -czf "$BACKUP_DIR/meshgateway_config_$DATE.tar.gz" \
    -C /opt/meshgateway/etc .

# 备份节点数据库
cp /opt/meshgateway/var/nodes.db "$BACKUP_DIR/nodes_$DATE.db"

# 清理旧备份 (保留7天)
find "$BACKUP_DIR" -name "*.tar.gz" -mtime +7 -delete
find "$BACKUP_DIR" -name "*.db" -mtime +7 -delete
```

### 9.2 恢复

```bash
# 恢复配置
sudo tar -xzf meshgateway_config_20240115.tar.gz -C /opt/meshgateway/etc

# 恢复节点数据库
sudo cp nodes_20240115.db /opt/meshgateway/var/nodes.db

# 重启服务
sudo systemctl restart meshgateway
```

## 10. 升级指南

### 10.1 升级步骤

```bash
# 1. 备份
sudo /opt/meshgateway/bin/backup.sh
```

```bash
# 恢复旧版本
sudo install -m 755 meshgateway.bak /opt/meshgateway/bin/meshgateway
sudo systemctl restart meshgateway
```

## 11. 安全加固

### 11.1 防火墙

```bash
# 仅允许本地访问 Socket (默认)
# 无需额外配置

# 如果需要远程访问，使用 SSH 隧道
ssh -L /run/meshgateway.sock:/run/meshgateway.sock user@host
```

### 11.2 AppArmor 配置

```bash
# /etc/apparmor.d/usr.bin.meshgateway
#include <tunables/global>

/opt/meshgateway/bin/meshgateway {
  #include <abstractions/base>
  #include <abstractions/nameservice>

  capability net_bind_service,

  /opt/meshgateway/bin/meshgateway mr,
  /opt/meshgateway/etc/** r,
  /opt/meshgateway/var/** rw,
  /run/meshgateway.sock w,
  /dev/ttyUSB* rw,
  /dev/ttyACM* rw,
  
  network unix,
}
```

### 11.3 限制资源

```ini
# /etc/systemd/system/meshgateway.service.d/limits.conf
[Service]
MemoryMax=50M
CPUQuota=25%
TasksMax=50
```
